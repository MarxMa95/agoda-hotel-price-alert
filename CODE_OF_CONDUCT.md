# Code of Conduct

## Our Pledge

We want this project to be a respectful, practical, and safe open-source space.
Contributors, maintainers, and users are expected to participate in a professional and constructive way.

## Our Standards

Examples of behavior that contributes to a positive environment include:

- Being respectful and patient
- Giving and receiving feedback constructively
- Focusing on reproducible technical issues
- Respecting project scope and safety constraints
- Acting in ways that support responsible use

Examples of unacceptable behavior include:

- Harassment, insults, or personal attacks
- Trolling or intentionally disruptive behavior
- Publishing private credentials, personal data, or webhook secrets
- Pressuring maintainers to add unsafe, abusive, or out-of-scope features
- Encouraging misuse of the software against platform rules or legal restrictions

## Enforcement Responsibilities

Project maintainers may remove, edit, or reject comments, issues, pull requests, and other contributions that violate this Code of Conduct or the repository's safety policies.

## Scope

This Code of Conduct applies within the GitHub repository and related issue or pull request discussions.

## Reporting

If you experience or observe unacceptable behavior, open a private security-style report when sensitive details are involved, or use the repository issue tracker for non-sensitive moderation concerns.

Also see:
- `SECURITY.md`
- `ACCEPTABLE_USE.md`
